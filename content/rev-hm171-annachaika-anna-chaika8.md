
Букву 'ц' я ввел второй, но в распечатке она стала первой.  
При этом введенная третьей буква 'к' в распечатке стала на третью позицию

Введенные буквы должны размещаться по алфавиту(й, к, ц) либо в порядке ввода(й, ц, к).

3. Но я же ввёл русские буквы

## ЗАМЕЧАНИЯ

**1. Нейминг**

- Только константы могут называться UPPER_SNAKE.

Константы это `static final` поля класса.  
Поля методов не являются константами 
```java

public class Game {
  static void main() throws IOException {
    int COUNT_ERRORS = 0;
    //...
  }
}

//ПРАВИЛЬНО:
public class Game {
  static void main() throws IOException {
    int errorCount = 0;
    //...
  }
}
```

- Соблюдай правила английского языка.  

"Счетчик ошибок" это "Error counter". Такую переменную можно назвать `errorCount`.

Словосочетание "count errors" означает "считать ошибки". 
Это глагол и поэтому переменная не может так называться.  
Название переменных и классов должны быть только существительными, методов- только глаголами.  
(не будем вдаваться в лингвистические тонкости названий переменных на префикс "has", которые могут называться как методы с аналогичным названием)   

- В названия не нужно вставлять частицы "Of", "The", "Are", "In" и т.д.

Это только делает названия более громоздкими.  
Если конечно там "Of" не используется в контексте valueOf()
```java
boolean letterInWord = isLetterInWord(inputLetter, oneWord);

//ПРАВИЛЬНО:
boolean isWordLetter = isWordLetter(inputLetter, oneWord);
```

- Вводящее в заблуждение название
```java
String oneWord = shuffledList.get(2);
String word = splitWord(oneWord);

//ПРАВИЛЬНО:
String text = shuffledList.get(2);
String word = splitWord(text);
```

- Здесь должно быть единственное число
```java
List<String> readsFile()

//ПРАВИЛЬНО:
List<String> readFile() 
```

- Название переменной должно объяснять, ЧТО ХРАНИТ переменная.  
А не те действия, которые с этой переменной кто-то когда-то будет делать
```java
String wordToEncrypt

//ПРАВИЛЬНО:
String word
```

Название "wordToEncrypt" может быть только у метода, который конвертирует какой-то "word" в какой-то "Encrypt".  
Где частица "To" означает процесс преобразования одного в другое.

- Название МЕТОДОВ пиши в настоящем времени, не пались на `irregular verbs`
```java
void foundLetter(String word, char inputLetter, StringBuilder encryptedWord)

//ПРАВИЛЬНО:
void findLetter(String word, char inputLetter, StringBuilder encryptedWord)
```

- Название (и комментарий) обманывают.

Метод не находит букву, он её открывает в маске слова(`encryptedWord`)
```java
public static void foundLetter(String word, char inputLetter, StringBuilder encryptedWord) { // Находим одинаковые буквы
  for (int i = 0; i < word.length(); i++) {
    if (inputLetter == word.charAt(i)) {
      encryptedWord.setCharAt(i, inputLetter);
    }
  }
}

//ПРАВИЛЬНО:
public static void openLetter(String word, char inputLetter, StringBuilder encryptedWord)
```

- Придерживайся правил английского языка.  

Сейчас название этого метода является существительным- "выбор символа"  
```java
char symbolSelection() 
```

Названия методов должны быть глаголом в настоящем времени в повелительном наклонении.  
При описании действия по правилам английского языка сначала ставится глагол.  
Правильно так:
```java
char selectSymbol()
```

- Используй общепринятые термины в названиях.  
Если что-то получаешь от юзера через ввод из клавиатуры, это "input"
```java
char symbolSelection() 

//ПРАВИЛЬНО:
char inputRussianLetter()  //потому что вводит именно русскую букву
```

- Переменная не может называться так же, как метод
```java
List<String> readsFile = readsFile();

//ПРАВИЛЬНО:
List<String> words = readsFile();
```

Потому что переменная должна быть существительным, а метод- глаголом.  
Исключение- переменные и методы, начинающиеся на "is" и "has"- они могут называться одинаково.

- Название обманывает.

Метод определяет не то, является ли входящий аргумент `symbolForStartGame` командой "Старт" или "Стоп".

Потому что метод с названием `isGameStartOrStop(char symbolForStartGame)` должен возвращать `true` и при "Старт" и при "Стоп".

Он говорит, что аргумент или является командой "Старт"(`true`) или не является ею(`false`)

```java
private static final Pattern PATTERN = Pattern.compile(REGEX);

//...
String input = scanner.nextLine().toLowerCase();
Matcher matcher = PATTERN.matcher(input);
if (matcher.matches()) {...}
```

**8. Исключения**

- Проверяемые исключения перехватывай в точке их возникновения и заменяй на непроверяемые. 

Проброс исключений в сигнатуре методов захламляет код.  
Поэтому заменяй проверяемые исключения на непроверяемые  
```java
public static List<String> readsFile() throws IOException { 
  Path path = Paths.get(PATH_TO_FILE);
  return Files.readAllLines(path);
}

//ПРАВИЛЬНО:

public static List<String> readFile() { 
  try {
    Path path = Paths.get(PATH_TO_FILE);
    return Files.readAllLines(path);
  } catch (IOException e) {
    throw new КакоетоНепроверяемоеИсключение(e);
  }
}
```
Изучи различия проверяемых и непроверяемых исключений  
*"ЧК", гл.7, "...проверяемые исключения"*

- Не бросай исключения в космос.

Метод main - последняя точка в программе, после которой исключение аварийно закрывает работу программы.  
Если ты знаешь, что в программе могут возникнуть исключения, то обработай их внутри программы,
а не бросай дальше в космос
```java
static void main() throws IOException {...} <-- Исключение летит в космос

//ПРАВИЛЬНО:
static void main() {...}  <-- Где-то внутри программы происходит перехват и обработка исключений
```

Программа всегда должна закрываться корректно.  
То есть, если что-то пошло не так, должно быть написано что-то типа "Блаблабла программа будет закрыта" и произойти корректное закрытие игры.  
Если через main пробрасывается исключение, значит в случае возникновения исключения программа будет закрыта аварийно с выбрасыванием непонятных красных сообщений в консоль.

**11. Печать картинок**

- В любом switch-case должен быть default.  
В данном случае default должен бросать исключение с сообщением, что такого номера картинки не существует
```java
switch (mistakes) {
  //...
  case 6:
    System.out.println(...);
    break;
  }
}

//ПРАВИЛЬНО:
switch (mistakes) {
  //...
  case 6:
    System.out.println(...);
    break;
  }
  default: //кинуть исключение 
}
```

Попытка распечатки несуществующей картинки это баг алгоритма.  
Его нужно как можно быстрее выявить и устранить.  
Поэтому в таком случае нужно сделать "быстрое падение" - кинуть исключение.

- Печать картинки виселицы через switch-case или if-elseif - индусский код.

Картинки нужно хранить в статическом массиве и печатать по номеру.  
Например, так
```java
public class HangmanRenderer {
  private static final String[] PICTURES = {
      """
    +----   
    |       
    |       
    |       
    ----- 
    """,
      """
     +---+   
     |   |   
     |   O   
     |       
     ----- 
     """
   ,
    // more pics
  };

  public static void render(int pictureNumber) {
    String picture = PICTURES[pictureNumber];

```

Желательно картинки и метод распечатки вынести в отдельный класс, чтобы в основном классе не смешивать логику и графику.

```java
}

private static boolean isGameOver() {
  return isWin() || isLose();  
}
```
Даже если метод состоит из одной строки, но при этом делает код программы читабельнее, то этот метод имеет право на существование.
