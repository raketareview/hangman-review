Введенные буквы должны размещаться по алфавиту(й, к, ц) либо в порядке ввода(й, ц, к).

## ЗАМЕЧАНИЯ

**1. Нейминг**

- Метод читает много слов, поэтому нужно написать во множественном числе
```java
ArrayList<String> readWordFromFile()

//ПРАВИЛЬНО:
ArrayList<String> readWordsFromFile()
```

- Лишние подробности.

Клиенту без разницы, читает этот класс слова из файла или откуда-то еще.  
Потому что факт того, что чтение происходит именно из файла, не требует от клиента принимать какиих-то действий.  
Например, передачи пути к файлу во входящие
```java
public class Dictionary {
  private static ArrayList<String> readWordFromFile() {...}
}

//ПРАВИЛЬНО ТАК:
public class Dictionary {
  private static ArrayList<String> read() {...}
}

//ТАК ТОЖЕ ПРАВИЛЬНО:
public class Dictionary {
  private static ArrayList<String> readWordFromFile(String filepath) {...}
}
```

- Название переменной обманывает
```java
System.out.println("Использованные буквы: " + usedWord);

//ПРАВИЛЬНО:
System.out.println("Использованные буквы: " + usedLetters);
```

**3. Комментарии**

Комментарии здесь часто не несут полезной нагрузки.  
Например, этот комент просто пересказывает название метода на русском языке
```java
//метод считывания слова с файла
private static ArrayList<String> readWordFromFile() 
```

Тут то же самое
```java
//метод показа приветственного меню для начала игры или ее завершения
private boolean startGameMenu()
```

А тут текст коментария противоречит названию метода.  
Судя по названию метода, он выполняет валидацию
```java 
//возвращает введенную пользователем букву
  private String letterValidation()
```

Когда в проекте много каментов, это плохо- пользы от них практически нет, они только забивают пространство и мешают читать код.  
В идеале, комментариев вообще не должно быть(кроме `TODO`), код должен объяснять сам себя через правильный нейминг и лаконичный код.

**4. Форматирование строк**

Если нужно печатать или создавать строку с более, чем одним подстановочным значением или значение вставляется внутрь сообщения, используй форматирование- тогда сразу будет виден весь шаблон
```java
System.out.println("Введите корректное значение: " + NEW_GAME + " или " + EXIT);

//ПРАВИЛЬНО:
System.out.printf("Введите корректное значение: %s или %s  %n", NEW_GAME, EXIT);
```

**5. Приоритет- читаемость кода**

Не экономь строчки в ущерб читаемости кода
```java
System.out.println("\nВведите " + NEW_GAME + ": Новая игра\n" + "Введите " + EXIT + ": Выйти из игры");

//ПРАВИЛЬНО:
System.out.println();
System.out.printf("Введите %s: Новая игра  %n",  NEW_GAME);
System.out.printf("Введите %s: Выйти из игры  %n", EXIT);
```

**6. Если в блоке if есть return(break, continue, throw, exit и т.д.), то else не пишется.**  
В этом случае неважно, будет else или нет, так как программа будет работать одинаково, а код без `else` будет выглядеть читабельней
```java
if (startGameMenu()) {
  startGameLoop();
} else {
  return;
}

//ПРАВИЛЬНО:
if (startGameMenu()) {
  startGameLoop();
} 
return;
```

**8. class Dictionary**

- Не используй секцию static. Это не дает ничего, кроме сложного для чтения кода 
```java
private static ArrayList<String> dictionary;

static {
  dictionary = readWordFromFile();
}

//ПРАВИЛЬНО:
private static ArrayList<String> dictionary = readWordFromFile();
```

- Нарушение конвенции кода. Сначала должны располагаться публичные методы, потом- приватные и протектед.

- Неожиданное поведение, нарушение принципа наименьшего удивления.

У класса есть поведение, которого от него не ожидаешь- он накладывает маску на слова
```java
public static String hideWord(String word) {
  return "_".repeat(word.length());
}
```
От класса с названием "Словарь" трудно ожидать такого поведения.  
От словаря мы ожидаем только получене слова.  
Вынеси этот метод в класс Hangman.

Принцип наименьшего удивления (Principle of Least Astonishment, POLA) в программировании гласит: 
```
Поведение компонента системы должно соответствовать ожиданиям большинства пользователей или разработчиков.
```

- Реакция на ошибки.

Если по указанному адресу не будет найден файл со словами, то программа аварийно вылетит и напечатает непонятные для юзера сообщения в консоль

**9. enum Gallow**

- Неправильное использование енамов.

Енамы нужно использовать как "константы на стероидах", в них не должно быть сложной логики.  
[Документация](https://docs.oracle.com/javase/8/docs/technotes/guides/language/enums.html) на enum сообщает:
```java
Итак, когда следует использовать перечисления (enums)? Всякий раз, когда вам нужен фиксированный набор констант.
```
Здесь нам не нужен набор констант, нам нужно просто получать картинку по ее номеру.  
Для этого нам достаточно одной константы в виде массива, а не набора отдельных констант.

У этого енама есть сложная прикладная логика, он скорее выполняет роль псевдокласса
```java
private String state;

//метод для перехода на следующее состояние виселицы
public Gallow nextState() {
  Gallow[] gallows = values();
  int next = this.ordinal() + 1;
  if (next >= gallows.length) {
    return this;
  }
  return gallows[next];
}
```
Получилось что-то вроде конечного автомата.

Лучше сделай простой класс-рендерер, который будет печатать картинку по ее номеру. 
Например, так
```java
public class HangmanRenderer {
  private static final String[] PICTURES = {
    """
    -----   
    |       
    |       
    |       
    |       
    ------- 
    """,
   """
     -----   
     |   |   
     |   O   
     |       
     |       
     ------- 
     """
   ,
   // more pics
  };

  public static void render(int pictureNumber) {  
    String picture = PICTURES[pictureNumber];
    System.out.println(picture);
```

- Метод должен вернуть одну букву. Поэтому возвращать он должен char
```java

//ПРАВИЛЬНО:
private char letterValidation()
```

- Нарушение правила одной операции.

Каждый метод должен выполнять только одну операцию.  
Этот метод делает разное- получает русскую букву и принимает ее, если она еще не была введена
```java
private String letterValidation()
```
*Мартин, "Чистый код", гл.3, "Правило одной операции"*

При чтении программы неочевидно, что происходит в алгоритме при вызове этого метода:
```java
char letter = letterValidation().charAt(0);
```

Можно улучшить название и тогда станет немного понятнее:
```java
char letter = inputUnusedRussianLetter()
```

Но, вообще, нельзя совмещать две разные операции в одном методе.  
Я бы советовал сделать метод, который получает только любую русскую букву, без доп. проверок:
```java
private char inputRussianLetter() {
  System.out.println("Введите букву: ");
  while (true) {
    //получить строку от юзера, 
    //если она не один символ- не принять и начать ввод заново
    //если строка один символ- преобразовать ее в char
    char letter = ...
    if(isRussianLetter(letter)) {
      return letter;
    }
    System.out.println("Введите корректное значение: (одну букву на русском)");
  }
}
```
А все остальное пусть делается за пределами этого метода.

- Нарушение правила одной операции.

Этот метод посредством вызова вспомогательного метода получает от юзера валидированную русскую букву и открывает букву в маске
```java
//возвращает строку с отгаданными или нет буквами
private String guessLetter() {
  char letter = letterValidation().charAt(0);  
  StringBuilder stringBuilder = new StringBuilder(hideWord);
  for (int i = 0; i < word.length(); i++) {
    if (word.charAt(i) == letter) {
      stringBuilder.setCharAt(i, letter);
    }
  }
  return stringBuilder.toString();
}
```
