## ЗАМЕЧАНИЯ

**1. Нейминг**

- Константы нужно писать стилем UPPER_SNAKE
```java
static final String[] HANGMANPICS

//ПРАВИЛЬНО:
static final String[] HANGMAN_PICS
```

- Не сокращай названия тогда, когда это сокращение не обусловлено очень длинными назщваниями
```java
Random rand = new Random();

//ПРАВИЛЬНО:
Random random = new Random();
```

- Венгерская нотация.

В названии переменных не пиши тип данных, к которым они относится.  
И вообще не употребляй венгерскую ноттацию.  
Название переменной должно отвечать на вопрос что хранит переменная, а не как хранит
```java
Set<Character> guessedLettersSet;

//ПРАВИЛЬНО:
Set<Character> guessedLetters;
```
*Oracle Java code conventions, part."Naming conventions"*

**2. Нарушение DRY**, магические буквы, числа, слова. Вводи константы 
```java

//ПРАВИЛЬНО:
private final static String START = "Y";
private final static String EXIT = "N";

if (answer.equalsIgnoreCase(START)) {...}
```

**3. Форматирование**

Если нужно печатать или создавать строку с более, чем одним подстановочным значением или значение вставляется внутрь сообщения, 
используй форматирование- тогда сразу будет виден весь шаблон.

```java
System.out.println("\n Буквы '" + letter + "' нет в слове!");

//ПРАВИЛЬНО:
System.out.println();
System.out.printf("Буквы '%с' нет в слове! %n", letter);
```

- Не передавай тернарники в аргументы методов- это тяжело читается

**5. Создавай вспомогательные методы, делай программу более простой и понятной**
```java

//ПРАВИЛЬНО:
char symbol = answer.charAt(0);

```

```java

//ПРАВИЛЬНО:
if (c == null) {
  return false;
```


**9. Нельзя завершать работу через exit()**

Метод не должен завершать работу программы через exit().  
Каждый метод и класс имеют право завершать только свою работу. Например, через return.  
Потому что методы и классы не должны знать логику работу более высоких слоев программы, у которых могут быть свои планы на тему того, когда и почему нужно завершать работу программы.

Кроме того, при выходе через `exit()` могут не закрыться некоторые ресурсы программы.

Часто `exit()` используют для упрощения алгоритма.  
Но здесь даже в этом нет необходимости- достаточно `exit()` заменить на обычный return
```java
public static void main() {
  while (true) {
    if (answer.equalsIgnoreCase("Y")) {
      //...
    } else {
      System.exit(0);  <-- Просто замени на return
    }
  }
}
```
