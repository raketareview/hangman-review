## ЗАМЕЧАНИЯ

**1. Нейминг**

- Не называй метод "run" если он не переопределяет `Runnable.run()`- название этого метода ассоциируется с этим стандартным интерфейсом и применением потоков.  
Здесь этот метод не имеет отношения к `Runnable.run()`
```java
public class GameController {
  public void run() {...}
  //...
}

//ПРАВИЛЬНО:
start() {...} //go(), execute(), perform(), поехали() 
```

- Венгерская нотация.

В названии переменных не пиши тип данных, к которым они относится.  
И вообще не употребляй венгерскую нотацию.  
Название переменной должно отвечать на вопрос что хранит переменная, а не как хранит
```java
String inputStr = input.get().toLowerCase();

//ПРАВИЛЬНО:
String command = input.get().toLowerCase();
```

*Oracle Java code conventions, part."Naming conventions"*

**2. Нарушение конвенции кода**

В любой ситуации выделяй тело блока скобочками, даже если тело состоит из одной строки.  
Исключение- метод equals(), там можно после if не выделять блоки скобочками
```java
if (lettersToGuess.contains(c))
  result.append("_");
else
  result.append(c);

for (Character c : word.toCharArray())
  lettersToGuess.add(c);

//ПРАВИЛЬНО:
if (lettersToGuess.contains(c)) {
  result.append("_");
} else {
  result.append(c);
}

for (Character c : word.toCharArray()) {
  lettersToGuess.add(c);
}
```

**3. Форматирование строк**

Если нужно печатать или создавать строку с более, чем одним подстановочным значением или значение вставляется внутрь сообщения, 
используй форматирование- тогда сразу будет виден весь шаблон
```java
out.println("Если Вы ошибетесь " + MAX_ERRORS + " раз - Вы проиграли!");
out.println("Поздравляю! В слове есть буква - " + letter + ".");

//ПРАВИЛЬНО:
out.printf("Если Вы ошибетесь %d раз - Вы проиграли!  %n", MAX_ERRORS);
out.printf("Поздравляю! В слове есть буква - %c.  %n", letter);
```

**9. class GameController**

+ 👍 Контроллер движка игры. Архитектура с движком это хорошо.

- Компилируй регулярные выражения. 

Если регулярное выражение используется многократно, как здесь, то нужно его откомпилировать.  
Компиляция регулярного выражения в объект `Pattern` с помощью `Pattern.compile()` позволяет повысить производительность при многократном использовании
```java
String inputStr = input.get().toLowerCase();
while (!inputStr.matches("[а-яё]")) {...}

//ПРАВИЛЬНО:
private static final String REGEX = "[а-яё]";
private static final Pattern PATTERN = Pattern.compile(REGEX);

//...
String inputStr = input.get().toLowerCase();
Matcher matcher = PATTERN.matcher(inputStr);
while (!matcher.matches()) {...}
```
Блох пишет:
```java
"Хотя String.matches — простейший способ проверки, соответствует ли строка регулярному выражению, 
он не подходит для многократного использования в ситуациях, критичных в смысле производительности." 
```
Здесь, конечно, нет критической ситуации с производительностью.  
Но здесь нет и необходимости просто так использовать неоптимальный алгоритм при работе с регулярными выражениями.  
*Блох "Java. Эффективное программирование", изд.3, гл.2.6*

- Нарушение SRP.

Диалог "Играть или выйти" должен происходить не в этом классе, а на уровень выше. В данном случае- в классе Main.

Для тренировки, сделай в проекте два Main'a.  
По запуску одного игра должена происходить так, как сейчас- по кругу через диалог "Играть или выйти".  
По запуску другого- игра должна выполниться один раз и без дилога "Играть или выйти".
